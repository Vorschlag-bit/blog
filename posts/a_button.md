## <a\> 태그와 <button\> 태그, 차이점이 뭘까?
이전까지는 Semantic한 HTML의 구조를 만드는 법에 대해서 배웠다. 이번에는 **사용자 기능**을 구현하는 태그에 관해서 배워보자!

프론트엔드 개발을 하면서 아무 생각없이 혹은 혼동해서 아래와 같은 코드들을 작성할 때가 있다.
```javascript
<!-- 잘못된 예시: 이동하는데 버튼을 씀 -->
<button onclick="location.href='/login'">로그인 페이지로 이동</button>

<!-- 잘못된 예시: 실행하는데 링크를 씀 -->
<a href="#" onclick="openModal()">팝업 열기</a>
```
두 코드 모두 사실 CSS로 **버튼** 모양을 만들면 겉보기엔 똑같아 보이지만, **브라우저와 스크린 리더**에겐 엄청난 큰 차이가 존재한다.  
이 글은 이 둘의 차이를 명확히 알고 가자는 의미.

### <a\> 태그 - "이동해라"
**Anchor**의 약자인 `<a>` 태그는 <strong>현재 페이지에서 다른 페이지로 이동할 때</strong> 사용한다.  
즉 **Navigation**의 의미를 갖고 있으며, 당연히 목적지인 `href`가 필수 속성이다.

- 사용 예시
    - 메뉴판의 메뉴 클릭(페이지 이동)
    - '더 보기' 링크 클릭
    - 같은 페이지 내에서 특정 위치로 점프(책갈피)

#### <a\> 태그만의 기능
마우스 오른쪽 버튼을 클릭했을 때, **"새 탭에서 열기"** 메뉴가 나온다면 그건 `<a>` 태그이다. 사용자는 링크를 눌러서 새 창에 띄울 수 있는 권리가 있는 셈이다.
`<button>`으로 이동 기능을 만든다면 이 기능을 구현하지 않겠다는 의미!

### <button\> - 눌러서 무언가를 "실행해라"
`<button>` 태그는 **페이지 이동 없이, 현재 화면에서 어떤 동작을 수행할 때** 사용한다.  
즉 **Action**의 의미를 갖고 있으며, 주요 속성으로 `type`이 있고 type에는 다양한 종류가 있다(`submit`, `button`, `disabled` 등)  

- 사용 예시
    - 로그인 폼 제출(submit)
    - 모달 창(팝업창) 열고 닫기
    - 다크 모드 토글 스위치
    - 좋아요 버튼

#### <button\> 태그만의 기능
`<button>` 태그는 `disabled` 기능을 사용할 수 있다. 예를 들어서, 필수 약관에 동의하지 않는다면 버튼을 비활성화하고 싶을 때가 있을 것이다.  
이때 `<a>` 태그를 사용한다면 `disabled` 기능을 사용할 수 없어서 별도의 자바스크립트 처리가 필요하다.

```HTML
<!-- 클릭하면 폼을 전송하거나 자바스크립트를 실행 -->
<button type="button" onclick="toggleMenu()">메뉴 열기</button>
```

### 해서는 안 되는 패턴들
#### 1. `<a href="#">` (가짜 링크)
버튼처럼 생겼으나 마우스를 갖다대면 손가락 커서가 나오게 하기 위해 사용하는 패턴이다.
```HTML
<a href="#" onclick="login()">로그인</a>
```
<a href="#" onclick="">로그인</a>  
위 로그인 링크를 클릭하는 순간 화면 최상단으로 점프하는 현상이 발생한다. 스크린 리더는 이를 단순히 "링크"라고만 읽어주는데, 클릭해도 페이지 이동은
전혀 없으므로 시각 장애인 이용자는 혼란에 빠질 것이다.

`<button>`을 사용하고 CSS로 `cursor: pointer`를 주도록 하자.

#### 2. `<div onclick="...">` (div 떡칠)
`<div>`에 클릭 이벤트를 걸어서 버튼처럼 사용하는 경우다.
```HTML
<div class="btn" onclick="submit()">전송</div>
```
키보드 접근성이 없는 코드다. `Tab`을 눌러서 포커스를 이동시킬 때 `div`는 선택되지 않고 지나치기 때문이다. 키보드만 사용하는 유저는 평생 이 버튼을 누를 수 없을 것이다..

### 접근성(Accessibility)과 키보드 조작
사실 이 둘을 구분하는 가장 큰 이유는 **웹 접근성** 때문이다.

| 구분 | Enter 키 | Space 키 | 스크린 리더 낭독 |
| :--- | :---: | :---: | :--- |
| **`<a>`** | 실행됨 (이동) | 스크롤 내려감 | "구글로 가기, **링크**" |
| **`<button>`** | 실행됨 | 실행됨 (눌림) | "전송, **버튼**" |

*   **`<a>` 태그:** 스페이스바를 누르면 화면이 아래로 스크롤 된다.
*   **`<button>` 태그:** 스페이스바를 누르면 버튼이 눌린다.

만약 버튼을 만들어야 하는데 `<a>` 태그를 썼다면, 키보드 사용자는 스페이스바를 눌렀을 때 버튼이 눌리는 게 아니라 화면이 덜컥거린다.

> 이걸 눌렀을 때 URL 주소가 바뀌는가?

*   **YES (주소 바뀜 / 새 탭 열기 가능):** -> `<a>` 태그
*   **NO (주소 안 바뀜 / 팝업 / 데이터 전송):** -> `<button>` 태그
