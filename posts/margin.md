## 여백을 조절하는 속성(margin, padding) 정리
두 박스 모델 사이의 여백은 `margin`, 한 박스 모델에서 테두리와 내용 사이의 여백은 `padding`이라고 한다.
margin과 padding은 웹 문서에서 여러 요소들을 배치할 때 매우 자주, 필수적으로 사용되는 요소이다.

이 글은 그러한 여백 관련 속성에 대해서 정리한 글.

### margin
<b>마진(margin)</b>은 요소 주변의 여백을 의미한다. 따라서 마진을 이용하면 요소와 요소 사이의 간격을 조절할 수 있다.  
테두리 속성과 마찬가지로 마진 속성 역시 박스 모델 4방향에 한꺼번에 스타일을 지정할 수도 있고, 4개 방향 각각에 스타일을
지정할 수도 있다. 방법은 테두리와 마찬가지로 margin 다음에 하이픈(-)을 넣고 위치를 나타내는 예악어인 top, right, bottom,
left를 사용해서 특정 방향을 지정하면 된다.

```CSS
p { margin: <크기> || <백분율> || auto }
```

`margin` 속성도 속성값을 하나만 지정하거나 2,3,4개를 지정할 수 있다. 값이 1개 라면 마진값을 4방향 모두 똑같이 지정하나,
여러 개라면 <b>top->right->bottom->left</b> 순(시계방향)으로 적용된다.

웹 브라우저의 개발자 도구 창을 이용하면 해당 요소의 마진을 쉽게 확인할 수 있다.

### margin 속성을 사용해 웹 문서 가운데 정렬
웹 문서에서 텍스트 요소를 배치할 때는 `text-align` 속성을 사용해서 정렬할 수 있다. 하지만 웹 문서 전체를 화면 중앙에 배치하려면 어떻게 해야 할까?  
이럴 때 사용하는 속성이 `margin`이다.

`margin` 속성을 사용해 화면 중앙에 내용을 배치하려면 우선적으로 배치할 요소의 너빗값이 정해져 있어야 한다. 그리고 `margin-left`와 `margin-right`의
속성값을 `auto`로 지정하면 CSS는 웹 브라우저 화면의 너비에서 요소 너빗값을 뺀 나머지 영역을 좌우 마진에 자동으로 할당한다.

### margin 중첩
박스 모델에서 마진을 지정할 때 주의해야 할 점이 있다. 요소를 세로로 배치할 경우 각 요소의 마진과 마진이 만나면 마진값이
큰 쪽으로 켭쳐지는 문제인데, 이를 <b>마진 중첩(margin overlap)</b>이라고 한다.

예를 들어 `<div>` 태그를 사용한 박스 영역 3개가 세로로 배치된다고 생각해보자. 박스 영역의 `margin` 속성값을 3개 모두 30px로 지정한다면
어떻게 될까? 앞서 배운 내용대로라면 3박스 모두 위/아래 30px씩, 각 박스 사이의 거리는 60px이 될 것이다.

하지만 실제로 결과를 보면 각 박스 사이의 거리는 30px로 되어 있다. 이렇게 된 이유는 여러 요소를 세로로 배치할 때 맨 위의 마진과 맨 아래 마진에 비해
중간에 있는 마진이 지나치게 커지는 걸 방지하기 위한 것이다. 마진 중첩은 <b>아래 마진과 위 마진이 서로 만날 때 더 큰 마진값으로 합쳐지는 것</b>이고
오른쪽 마진과 왼쪽 마진이 만날 경우에는 중첩되지 않는다.

### padding
<b>패딩(padding)</b>이란 <b>콘텐츠 영역과 테두리 사이의 여백</b>을 말한다. 다시 말해 테두리 안쪽의 여백이라고 생각하면 된다.  
패딩과 마진은 여백이 어느 위치에 있느냐만 다를 뿐 박스 모델에서 패딩 지정은 마진 지정 방법과 비슷하다.

`padding` 속성은 4개 방향으로 한꺼번에 여백을 줄 수 있고, 하이픈 + 위치 예약어로 특정 방향에만 padding을 줄 수도 있다.

```CSS
/* 4방향 모두 padding 넣기 */
p { padding: 20px }
/* 4방향 다른 padding 넣기 */
p { padding: 20px 10px 20px 10px }
```

- **`margin` (외부 여백)**: 요소의 바깥쪽 여백을 의미하며 요소 간의 간격을 조절함. 속성값이 여러 개일 경우 `top` → `right` → `bottom` → `left` (시계 방향) 순서로 적용됨.
- **가운데 정렬**: 요소의 너비(`width`)가 정해져 있을 때, 좌우 마진(`margin-left`, `margin-right`)을 `auto`로 설정하면 요소를 중앙에 배치할 수 있음.
- **마진 중첩 (Margin Overlap)**: 세로로 배치된 요소들의 상하 마진이 만날 경우, 두 값이 합쳐지는 것이 아니라 **더 큰 값 하나만 적용**되는 현상임. (좌우 마진은 중첩되지 않음)
- **`padding` (내부 여백)**: 콘텐츠와 테두리(border) 사이의 안쪽 여백을 의미함. 마진과 사용법이 동일하며 4방향을 개별적으로 혹은 한꺼번에 지정할 수 있음.
