## CSS(Cascading Style Sheet)
스타일 시트라는 용어는 CSS로도 잘 알려져 있다. 여기서 왜 단순히 스타일 시트가 아니라 <b>Cascading</b>이 붙어있는 걸까?  
그건 스타일 시트에선 우선순위가 존재하고 위 -> 아래로 적용되기 때문이다. 즉, CSS는 우선순위가 있는 스타일 시트라는 의미이다.

따라서 CSS에선 웹 요소에 둘 이상의 스타일을 적용하면 우선순위에 따라 스타일이 결정된다.  
스타일이 충돌하지 않기 위해선 2가지 원칙을 반드시 준수해야 한다.

> <b>스타일 우선순위</b>: 스타일 규칙의 중요도와 적용 범위에 따라 우선순위가 결정되고, 그 우선순위에 따라 위에서 아래로 스타일을 적용한다.  
> <b>스타일 상속</b>: 태그의 포함 관계에 따라 부모 요소의 스타일을 자식 요소로, 위에서 아래로 전달한다.

이렇게 스타일 충돌을 방지하는 스타일 우선순위와 스타일 상속에 대해서 좀 더 알아보자.
### 스타일 우선순위
스타일 우선순위는 Cascading에서 가장 중요하다. 이때 우선순위란 어떤 스타일을 먼저 적용할지 결정하는 규칙을 말한다.  
우선순위는 3가지 개념에 따라서 결정된다.

#### 기본 중요도
웹 브라우저에 내용을 표시할 때 단순히 CSS 코드의 스타일만 적용되는 건 아니다. <b>컴퓨터 사용자가 지정한 스타일</b>과 <b>웹 문서를 제작한 제작자 스타일</b> 그리고
<b>웹 브라우저 기본 스타일</b> 이렇게 3가지 스타일을 함께 적용한다.

<div class="mermaid">
    flowchart TD
        A(1. 사용자 스타일)
        A --> K(2. 제작자 스타일)
        K -->D(3. 브라우저 기본 스타일)
</div>

우선순위는 <b>사용자 스타일 > 제작자 스타일 > 브라우저 기본 스타일</b>이다.

#### 적용 범위
중요도가 같은 스타일이라면 스타일 적용 범위에 따라 우선순위가 결정된다. 스타일 적용 범위가 좁을 수록, 즉 필요한 요소에만 정확히 적용할 스타일일수록
우선순위가 높아진다.  
단, 스타일 규칙에 <b>!important</b>를 붙이면 그 스타일은 다른 스타일보다 우선 순위가 높아진다.

```CSS
/* 이렇게 하면 모든 요소(*)에서 둥근 모서리가 제거된다. */
* {
    border-radius: 0 !important;
}
```

    flowchart TD
        A(1. !important)
        A --> K(2. 인라인 스타일)
        K -->D(3. id 스타일)
        D -->E(4. 클래스 스타일)
        E -->F(5. 타입 스타일)
</div>

</a>에서 상세한 설명을 찾아볼 수 있다.

#### 소스 코드 작성 순서
스타일 시트에서 중요도와 적용 범위까지 같다면 마지막으로 스타일을 정의한 <b>코드 순서</b>에 따라 우선순위가 결정된다.  
코드에서 <b>나중에 작성한 스타일이 먼저 작성한 스타일을 덮어쓴다</b>. 

```HTML
<style>
    p {
        color: black;
    }
    p {
        color: red;
    }
</style>
<p>아무말 대잔치</p>
```
위와 같은 코드가 있을 경우 스타일 중요도(제작자)와 적용 범위(태그)까지 모두 같기 때문에 나중에 적은 `color: red`가 적용된다.

### 스타일 상속
웹 문서에서 사용하는 여러 태그는 완벽한 <b>트리(Tree)구조</b>를 이루고 있다. 즉, 부모와 자식 관계로 표현이 가능하다.  
상위 태그(포함하는 태그)를 부모 요소, 하위 태그(포함된 태그)를 자식 요소라고 하며, 스타일 시트에서 자식 요소에 별도의 스타일이
지정되지 않는다면 부모 요소의 스타일이 자식에게 전달되는데, 이를 <b>스타일 상속</b>이라고 한다.

예를 들어 `<body>` 태그는 웹 문서에서 사용되는 분문 속 모든 태그의 부모 태그이다. 따라서 `<body>` 태그에 글자 크기나 정렬을 준다면 그 스타일은 해당 웹 문서의
전체에 적용이 되는 것이다.

- **캐스케이딩(Cascading)**: CSS는 단순한 나열이 아니라 **우선순위**가 존재하며, 규칙에 따라 위에서 아래로 스타일이 적용된다는 의미를 가진다.
- **스타일 우선순위(Priority)**: 여러 스타일이 충돌할 때 다음 세 가지 기준에 따라 적용 여부가 결정된다.
    1.  **기본 중요도**: 사용자 스타일 > 제작자 스타일 > 브라우저 기본 스타일 순서로 우선순위가 높음.
    2.  **적용 범위(명시도)**: `!important` > 인라인 > ID > 클래스 > 타입 선택자 순이며, 적용 범위가 좁고 구체적일수록 우선순위가 높음.
    3.  **소스 코드 작성 순서**: 중요도와 범위가 모두 같다면, <b>나중에(가장 아래에) 작성된 스타일</b>이 이전 스타일을 덮어씀.
- **스타일 상속(Inheritance)**: HTML의 계층(트리) 구조에 따라, 부모 요소에 지정된 스타일 속성이 자식 요소에게 자동으로 전달되는 특성.
