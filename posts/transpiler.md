## 자바스크립트는 계속 진화한다

자바스크립트(ECMAScript)는 매년 새로운 문법과 기능이 추가되며 빠르게 발전하고 있다.
하지만 개발자가 최신 문법(`ES6+`)으로 코드를 작성하더라도, 사용자가 쓰는 <b>구형 브라우저(IE 등)</b>는 이를 이해하지 못해 에러를 뿜어낼 수 있다.

이 문제를 해결하기 위해 등장한 것이 바로 <b>트랜스파일러(Transpiler)</b>와 <b>폴리필(Polyfill)</b>이다. 둘 다 "호환성"을 위해 존재하지만, 그 역할과 동작 방식은 완전히 다르다.

---

## 1. 트랜스파일러 (Transpiler)

### 1-1. 정의
트랜스파일러는 **한 프로그래밍 언어로 작성된 코드를 다른 언어(또는 다른 문법)로 변환해 주는 도구**다.

*   <b>컴파일러(Compiler)</b>: 고급 언어(C, Java) → 기계어/바이트코드 (저수준 언어)
*   <b>트랜스파일러(Transpiler)</b>: 고급 언어(ES6, TS) → 고급 언어(ES5, JS) (동등한 수준)

### 1-2. 대표적인 예시
1.  **Babel**: 최신 자바스크립트(ES6+) 문법을 구형 브라우저도 이해할 수 있는 ES5 문법으로 변환한다.
    *   화살표 함수 `() => {}` → `function() {}`
    *   `const`, `let` → `var`
2.  **TypeScript (TSC)**: 브라우저는 TypeScript를 이해하지 못하므로, 이를 표준 JavaScript 코드로 변환한다.
3.  **JSX 변환**: React의 JSX 문법(`<Div>...`)을 `React.createElement(...)` 함수 호출로 변환한다.

### 1-3. 왜 쓰는가?
트랜스파일러가 없다면 개발자는 구형 브라우저 지원을 위해 **최신 문법의 편리함과 생산성**을 포기하고 옛날 문법으로만 코드를 짜야 한다. (TypeScript의 타입 시스템도 사용 불가)

---

## 2. 폴리필 (Polyfill)

### 2-1. 정의
폴리필은 <b>브라우저가 지원하지 않는 최신 기능(API)을 구현하여 메워주는 코드 조각(플러그인)</b>이다.
트랜스파일러가 <b>"문법(Syntax)"</b>을 바꾼다면, 폴리필은 <b>"기능(Feature)"</b>을 추가한다.

### 2-2. 트랜스파일러와의 결정적 차이
가장 큰 차이는 **변환 시점**과 **대상**이다.

| 구분 | 트랜스파일러 (Babel 등) | 폴리필 (core-js 등) |
| :--- | :--- | :--- |
| **실행 시점** | **빌드 타임** (코드 작성 후 변환) | **런타임** (브라우저에서 실행될 때) |
| **대상** | 문법 (Syntax) | API, 객체, 메서드 |
| **예시** | 화살표 함수, 클래스, 비구조화 할당 | `Promise`, `Map`, `Array.includes` |

### 2-3. 왜 폴리필이 필요한가?
트랜스파일러는 문법만 바꿀 뿐, **없던 기능을 창조하지는 못한다.**
예를 들어 `Promise`는 ES6에 추가된 새로운 객체다. Babel은 `Promise`라는 단어를 다른 걸로 바꿀 수 없다. 따라서 구형 브라우저에 `Promise`라는 객체 자체가 정의되어 있지 않다면, **런타임에 직접 `window.Promise`를 구현해서 넣어주는(충전해 주는) 폴리필**이 필요하다.


> **트랜스파일러**: 한국어 책을 영어로 번역해 주는 번역가. (문법 변환)  
> **폴리필**: 영어 사전에 없는 신조어(Promise)가 나오면, 그 단어의 뜻풀이를 책 맨 뒤에 추가해 주는 부록. (기능 정의 추가)

## 3. 요약: 모던 웹 개발의 필수품

우리가 `React`, `Vue` 등으로 편하게 개발할 수 있는 이유는 이 도구들이 뒤에서 묵묵히 일하고 있기 때문이다.

1.  **개발자**: 최신 문법(ES6+), TypeScript, JSX로 편하게 코딩한다.
2.  **트랜스파일러 (Build Time)**:
    *   TypeScript → JavaScript 변환
    *   JSX → JS 함수 변환
    *   화살표 함수 → 일반 함수 변환
3.  **폴리필 (Run Time)**:
    *   브라우저가 실행될 때, `Promise`나 `Array.from` 같은 최신 기능이 없으면 직접 구현체를 주입해서 돌아가게 만든다.

이 두 가지가 합쳐져야 비로소 <b>"개발 편의성(DX)"</b>과 <b>"크로스 브라우징(사용자 호환성)"</b>을 모두 잡을 수 있다.

- **트랜스파일러**: 문법을 변환한다. (예: Babel, TypeScript) 빌드 타임에 작동하며, 최신 문법을 구형 문법으로 바꿔준다.
- **폴리필**: 기능을 추가한다. (예: core-js) 런타임에 작동하며, 브라우저에 없는 최신 API(`Promise` 등)를 구현해 준다.
- **결론**: 최신 웹 개발 환경을 구축하려면 두 가지 모두 적절히 설정하여 사용해야 한다.
