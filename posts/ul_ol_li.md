## <ul\>, <ol\>, <li\> 태그의 특성에 대해서 학습
이전에 `<a>`와 `<button>` 태그를 통해서 사용자의 행동(이동/실행)을 구분하는 방법에 대해서 배웠다.  

이 글에서는 <b>정보의 그룹핑(Grouping)</b>, 즉 목록을 만드는 태그에 대해서 학습한다.  
그냥 `<p>` 태그에 `classID`로 1,2,3 적으면 되는 거 아닌가?라고 생각할 수도 있겠지만 웹 접근성을 고려한
<b>Semantic한 HTML 파일</b>을 만드는 것은 의무적이므로 그룹핑에 왜 <code><ol\></code>, <code><ul\></code>, <code><li\></code>을 사용하는지 알 필요가 있다!

### 1. 목록 태그 3대장: <ol\>, <ul\>, <li\>
이 셋은 거의 세트로 다닌다고 생각하면 된다. 부모(`<ol>`, `<ul>`)와 자식(`<li>`)관계이기 때문이다.

#### 1-1. <ol\>: 순서가 있는 목록
순서(랭킹, 단계)가 중요한 항목을 묶을 때 사용하는 태그이다. 기본 스타일로 <b>숫자(1,2,3...)</b>가 자동으로 붙게 된다.  
목차나, 레시피와 같이 명확한 순서가 존재하는 걸 표현할 때 부모 태그로서 `<li>`를 감싸면 된다.
```HTML
<ol>
  <li>물 550ml를 끓인다.</li>
  <li>면과 스프를 넣는다.</li>
  <li>4분간 더 끓인다.</li>
</ol>
```

#### 1-2. <ul\>: 순서가 없는 목록
순서가 종요하지 않은 항목을 묶을 때 사용하는 태그이다. 기본 스타일로 <b>점(•)</b>이 자등으로 붙게 된다.  
단순한 목록(내 블로그 글 같이)이나 메뉴들을 보여줄 때 부모 태그로서 `<li>`를 감싸면 된다.
```HTML
<ul>
  <li>사과</li>
  <li>바나나</li>
  <li>딸기</li>
</ul>
```

#### 1-3. <li\>: 목록의 항목
`<ol>`이나 `<ul>`의 직계 자식 태그는 <b>반드시 `<li>` 태그여야만 한다</b>. `<div>`나 `<p>`가 들어갈 수 없다.

### 2. 왜 이 태그들을 사용해야 할까?
앞에서도 짧게 말했지만, 역시나 <b>웹 접근성</b>때문이다.  
시각 장애인이 사용하는 스크린 리더를 통해서 만약 단순히 태그 안의 글자로 나열된 리스트를 본다면 줄 글로 리스트의 모든 항목을 읊을 뿐 이게
연관 관계가 있는 건지 전혀 알 수가 없다.
```HTML
<p>1. 사과</p>
<p>2. 배</p>
<p>3. 포도</p>
```
이런 식으로 작성된 HTML 태그는 스크린 리더가 그냥 "일 점 사과, 이 점 배, 삼 점 포도"로 읽을 뿐이라 <b>그룹으로서의 의미</b>를 전혀 가지지 않는다.  

반면 `<ul>`이나 `<ol>`를 통해서 작성된 아래와 같은 코드는 <b>목록, 항목 3개. 사과, 3개 중 1번째. 배, 3개 중 2번째. 포도, 3개 중 3번째. 목록 끝</b>과
같이 <b>전체 항목이 몇 개인지, 지금 읽는 게 몇 번째인지, 어디서 목록이 끝나는지</b>를 명확하게 알려준다. 이 덕분에 정보의 구조를 파악하는 데에 엄청 큰 도움이 될 것이다.

### 3. <nav\> 태그는 목록일까?
우리가 흔히 보는 웹사이트에선 상단 메뉴는 보통 <b>가로로 배열</b>되어 있다.  
디자인 상, 점(•)이 없어서 헷갈릴 수도 있겠지만 의미상 명백히 <b>메뉴의 목록</b>이기 때문에 `<ul>`를 사용하는 게 알맞다.  
물론 점을 없애는 것은 단순히 CSS의 영역이다. (`list-style: none`)

<div class="flex items-center gap-2">
    <svg class="w-10 h-10 text-gray-800" fill="none" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24"> <path d="M5 3H3v18h18V3H5zm0 2h14v14H5V5zm4 7H7v2h2v2h2v-2h2v-2h2v-2h2V8h-2v2h-2v2h-2v2H9v-2z" fill="currentColor"/> </svg>
    <span class="font-bold text-2xl">
        글 요약
    </span>
</div>

1. 순서 없으면 `<ul>`, 순서 있으면 `<ol>`을 쓴다.
2. 항목은 무조건 `<li>`로 감싼다.
3. 목록 태그를 써야 스크린 리더가 <b>"항목 개수와 순서"</b>를 안내해 준다.
4. 네비게이션 메뉴도 `<ul>`이다. CSS로 디자인을 바꾼 것일 뿐이다.
